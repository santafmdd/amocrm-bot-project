# amoCRM API Bootstrap (External Integration)

## Purpose
Minimal local bootstrap to obtain and verify amoCRM API access tokens.
No analyzer business logic, no weekly/analytics writer changes.

## New module
- `src/amocrm_auth/config.py`
- `src/amocrm_auth/state_store.py`
- `src/amocrm_auth/oauth_client.py`
- `src/amocrm_auth/local_callback_server.py`
- `src/amocrm_auth/bootstrap.py`
- `src/amocrm_auth/exchange_code.py`
- `src/amocrm_auth/smoke_test.py`

## Config
1. Copy `config/amocrm_auth.example.json` to `config/amocrm_auth.local.json`.
2. Fill values:
- `base_domain`: `<your_account>.amocrm.ru`
- `client_id`: external integration client id
- `client_secret`: external integration client secret
- `redirect_uri`: local callback URL (default `http://127.0.0.1:18080/oauth/callback`)

You can also override by env:
- `AMOCRM_BASE_DOMAIN`
- `AMOCRM_CLIENT_ID`
- `AMOCRM_CLIENT_SECRET`
- `AMOCRM_REDIRECT_URI`
- `AMOCRM_AUTH_STATE_FILE`

## What to put into amoCRM external integration
- Redirect URI: `http://127.0.0.1:18080/oauth/callback`
- Integration name: neutral technical name (no AI/LLM/GPT wording)
- Client ID / Secret: generated by amoCRM integration page

## OAuth bootstrap flow
Run from project root:

```powershell
python -m src.amocrm_auth.bootstrap --config config/amocrm_auth.local.json
```

What happens:
1. local callback HTTP server starts;
2. auth URL is printed (and browser opens by default);
3. callback receives `code/state/referer`;
4. code is exchanged for token set;
5. state is saved to `workspace/amocrm_auth/state.json`;
6. smoke-test request to `/api/v4/account?with=users` is executed.

## Manual long-lived token mode
If you obtained long-lived token separately:

```powershell
python -m src.amocrm_auth.bootstrap --config config/amocrm_auth.local.json --base-domain youraccount.amocrm.ru --set-long-lived-token <TOKEN>
```

## Smoke test only
```powershell
python -m src.amocrm_auth.smoke_test --config config/amocrm_auth.local.json
```

Optional refresh when token is expired and refresh token exists:
```powershell
python -m src.amocrm_auth.smoke_test --config config/amocrm_auth.local.json --refresh-if-needed
```

## State files
- Config template: `config/amocrm_auth.example.json`
- Local config (you create): `config/amocrm_auth.local.json`
- Runtime state: `workspace/amocrm_auth/state.json`

## Notes
- No secrets are hardcoded in source.
- Callback server is local-only by default (`127.0.0.1`).
- Keep local config/state out of VCS if they contain credentials.
